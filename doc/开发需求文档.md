# Component 模块开发需求文档

## 📋 模块概述

**模块名称**：Weline_Component  
**模块版本**：1.0.0  
**开发日期**：2024年  
**文档版本**：1.0

## 🎯 功能需求

### 1. 组件扫描功能
- **需求描述**：扫描系统中所有实现了 `ComponentInterface` 接口的 Block 类
- **技术要求**：
  - 自动扫描所有已安装模块的 `Block/` 目录
  - 识别实现了 `ComponentInterface` 的类
  - 提取组件信息（模块名、类名、命名空间、分类、文档等）
  - 支持递归扫描子目录

### 2. 组件库管理界面
- **需求描述**：提供后台管理界面，展示所有可用组件
- **功能要求**：
  - 组件列表展示
  - 按模块筛选
  - 按分类筛选
  - 关键词搜索
  - 组件详情查看
  - 统计信息展示

### 3. 组件分类功能
- **需求描述**：自动根据命名空间提取组件分类
- **技术要求**：
  - 从命名空间路径提取分类信息
  - 支持默认分类"其他"
  - 按分类组织组件列表

## 🔧 技术方案

### 1. 架构设计
- **Service层**：`ComponentScanner` 负责组件扫描逻辑
- **Controller层**：`Components` 负责后台管理界面
- **Model层**：`Component` 模型（预留，用于未来扩展）
- **Interface**：`ComponentInterface` 定义组件接口规范

### 2. 核心类设计

#### ComponentScanner
- `scanAllComponents()`: 扫描所有组件
- `scanDirectory()`: 递归扫描目录
- `checkComponentClass()`: 检查类是否实现接口
- `groupByCategory()`: 按分类组织组件

#### Components Controller
- `index()`: 组件库首页，支持筛选和搜索
- `detail()`: 组件详情（AJAX接口）

### 3. 数据流程
1. 获取所有已安装模块列表
2. 遍历每个模块的 `Block/` 目录
3. 扫描 PHP 文件，检查是否实现 `ComponentInterface`
4. 提取组件信息并组织数据
5. 在前端展示组件列表
